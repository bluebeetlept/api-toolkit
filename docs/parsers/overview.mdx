Parsers are the building blocks that power the [QueryBuilder](/query-builder/overview). While the QueryBuilder is the recommended way to use them, each parser is available individually for fine-grained control.

## Available parsers

| Parser | Query parameter | Example |
|--------|----------------|---------|
| [FilterParser](/parsers/filters) | `filter` | `?filter[status]=active` |
| [SortParser](/parsers/sorting) | `sort` | `?sort=-created_at,name` |
| [IncludeParser](/parsers/includes) | `include` | `?include=category,supplier` |
| [FieldParser](/parsers/sparse-fieldsets) | `fields` | `?fields[products]=name,code` |
| [PageParser](/parsers/page) | `page` | `?page[number]=2&page[size]=20` |

## Direct usage

Each parser can be used independently:

```php
use Eufaturo\ApiToolkit\Parsers\FilterParser;
use Eufaturo\ApiToolkit\Parsers\Filters\ExactFilter;

$parser = new FilterParser([
    'status' => new ExactFilter(),
]);

$parser->apply($request, $query);
```

This is useful when you need to apply parsing logic outside the context of a QueryBuilder, for example in a custom query scope or a service class.
