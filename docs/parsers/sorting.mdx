---
title: Sorting
description: Parse and apply sort query parameters
---

The `SortParser` handles the `?sort` query parameter following the [JSON:API sorting specification](https://jsonapi.org/format/#fetching-sorting).

## Format

```
?sort=name              → ORDER BY name ASC
?sort=-created_at       → ORDER BY created_at DESC
?sort=-created_at,name  → ORDER BY created_at DESC, name ASC
```

Prefix a field with `-` to sort descending. Multiple fields are comma-separated.

## Usage

```php
use Eufaturo\ApiToolkit\Parsers\SortParser;

$parser = new SortParser(
    allowed: ['name', 'created_at', 'code'],
    default: '-created_at',
);

$parser->apply($request, $query);
```

## Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `allowed` | `list<string>` | Fields that can be sorted on. Empty = allow all. |
| `default` | `string\|null` | Default sort when no `?sort` is provided. Prefix `-` for desc. |

## Disallowed fields

Sort fields not in the `allowed` list are silently ignored:

```php
$parser = new SortParser(allowed: ['name']);

// ?sort=secret_column → ignored, no sort applied
```

## Default sort

When no `?sort` parameter is provided, the `default` is used:

```php
$parser = new SortParser(
    allowed: ['name', 'created_at'],
    default: '-created_at',
);

// No ?sort → ORDER BY created_at DESC
```

If `default` is `null` and no `?sort` is provided, no ordering is applied.
