title: Overview
description: Automatic JSON:API error responses for exceptions
---

The exception handler automatically converts Laravel exceptions into JSON:API compliant error responses.

## Error format

All exceptions are rendered as JSON:API errors:

```json
{
  "errors": [
    {
      "status": "404",
      "code": "invalid_request_error",
      "title": "Not Found",
      "detail": "No such product: 'abc-123'"
    }
  ]
}
```

## Handled exceptions

| Exception | Status | Code |
|-----------|--------|------|
| `AuthenticationException` | 401 | `invalid_request_error` |
| `ValidationException` | 422 | `validation_error` |
| `QueryException` | 400 | `invalid_request_error` |
| `LazyLoadingViolationException` | 400 | `invalid_request_error` |
| `RouteNotFoundException` | 404 | `invalid_request_error` |
| `ModelNotFoundException` | 404 | `invalid_request_error` |
| `NotFoundHttpException` | 404 | `invalid_request_error` |
| `MethodNotAllowedHttpException` | 404 | `invalid_request_error` |
| `IdempotencyException` | varies | `idempotency_error` |
| `HttpExceptionInterface` | varies | `invalid_request_error` |
| Domain exceptions (configurable) | 400 | `invalid_request_error` |
| All other exceptions | 500 | `api_error` |

## Validation errors

Validation exceptions produce one error per field, with JSON pointers:

```json
{
  "errors": [
    {
      "status": "422",
      "code": "validation_error",
      "title": "Validation Error",
      "detail": "The name field is required.",
      "source": { "pointer": "/name" }
    },
    {
      "status": "422",
      "code": "validation_error",
      "title": "Validation Error",
      "detail": "The email must be a valid email address.",
      "source": { "pointer": "/email" }
    }
  ]
}
```

Nested fields use JSON pointer notation: `billing.address.city` becomes `/billing/address/city`.

## Debug mode

When `APP_DEBUG=true`, errors include a `meta.debug` object:

```json
{
  "errors": [
    {
      "status": "500",
      "code": "api_error",
      "title": "Internal Server Error",
      "detail": "...",
      "meta": {
        "debug": {
          "line": 42,
          "file": "/app/Services/ProductService.php",
          "class": "RuntimeException",
          "trace": [...]
        }
      }
    }
  ]
}
```

<Warning>
  Debug information is never included when `APP_DEBUG=false`.
</Warning>

## Model not found

When a model is not found via route model binding, the error message is human-friendly:

```json
{
  "errors": [
    {
      "status": "404",
      "code": "invalid_request_error",
      "title": "Not Found",
      "detail": "No such product: 'abc-123'"
    }
  ]
}
```
