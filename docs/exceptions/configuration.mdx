## Setup

Configure the exception handler in your application's bootstrap:

```php bootstrap/app.php
use Eufaturo\ApiToolkit\Exceptions\ConfigureExceptionHandler;

return Application::configure(basePath: dirname(__DIR__))
    ->withExceptions(new ConfigureExceptionHandler())
    ->create();
```

## Domain exceptions

Register your domain exception classes so they are rendered as 400 Bad Request instead of 500:

```php config/api-toolkit.php
'exceptions' => [
    'domain' => [
        \App\Exceptions\InsufficientStockException::class,
        \App\Exceptions\InvalidOrderStateException::class,
    ],
],
```

Domain exceptions are rendered as:

```json
{
  "errors": [
    {
      "status": "400",
      "code": "invalid_request_error",
      "title": "Bad Request",
      "detail": "Insufficient stock for product Widget"
    }
  ]
}
```

The `detail` is taken from the exception's `getMessage()`.

## Don't report

Prevent certain exceptions from being reported to your error tracker (e.g., Sentry, Bugsnag):

```php config/api-toolkit.php
'exceptions' => [
    'dont_report' => [
        \App\Exceptions\InsufficientStockException::class,
        \App\Exceptions\RateLimitedException::class,
    ],
],
```

These exceptions are still rendered as JSON:API errors. They are simply not reported.

<Note>
  Duplicate exception reporting is automatically prevented regardless of this configuration.
</Note>
