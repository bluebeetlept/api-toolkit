Resources are the core building block of API Toolkit. They define how your data is serialized into [JSON:API resource objects](https://jsonapi.org/format/#document-resource-objects).

## Basic structure

```php
use Eufaturo\ApiToolkit\Resources\Resource;
use App\Models\Product;

final class ProductResource extends Resource
{
    protected string $model = Product::class;

    public function attributes(Product $product): array
    {
        return [
            'name' => $product->name,
            'code' => $product->code,
        ];
    }

    public function self(Product $product): string | null
    {
        return route('api.v1.products.show', $product);
    }
}
```

This produces:

```json
{
  "type": "products",
  "id": "abc-123",
  "attributes": {
    "name": "Widget",
    "code": "W01"
  },
  "links": {
    "self": "https://api.example.com/v1/products/abc-123"
  }
}
```

## Generating resources

Use the artisan command to scaffold a new resource:

```bash
php artisan api-toolkit:make-resource ProductResource
```

This creates `app/Http/Resources/ProductResource.php` with the model auto-guessed from the name (`Product`):

```php
final class ProductResource extends Resource
{
    protected string $model = Product::class;

    public function attributes(Product $product): array
    {
        return [
            //
        ];
    }
}
```

### Options

```bash
# Explicit model class
php artisan api-toolkit:make-resource ProductResource --model=App\\Models\\Product

# Plain resource without a model (uses $type instead of $model)
php artisan api-toolkit:make-resource DateResource --plain
```

## Model and type

### With a model

Set `$model` to an Eloquent model class. The JSON:API `type` is automatically derived from the model's table name:

```php
final class ProductResource extends Resource
{
    protected string $model = Product::class;
    // type = "products" (from Product's table name)
}
```

### Without a model

For non-model data, set `$type` directly and type-hint whatever you need:

```php
final class DateResource extends Resource
{
    protected string $type = 'dates';

    public function attributes(CarbonInterface $date): array
    {
        return [
            'human' => $date->diffForHumans(),
            'timestamp' => $date->timestamp,
        ];
    }
}
```

### Overriding the type

You can set both `$model` and `$type`. The explicit `$type` takes precedence:

```php
final class LegacyProductResource extends Resource
{
    protected string $model = Product::class;
    protected string $type = 'legacy-products'; // overrides table name
}
```

### Type resolution order

1. `$type` if explicitly set
2. Global type resolver (if registered)
3. `$model` table name if set
4. Runtime model's table name (for Eloquent models passed at runtime)
5. Snake-cased class name as fallback

### ID resolution order

1. Global ID resolver (if registered)
2. `$model->public_id` (for Eloquent models)
3. `$model->getKey()` (for Eloquent models)
4. `$model->id` or `$model->public_id` (for plain objects)

## Global resolvers

Register global callbacks to customize how IDs and types are derived across all resources. Useful for projects that use UUIDs, hashids, or custom naming conventions.

Register in a service provider:

```php
use Eufaturo\ApiToolkit\Resources\Resource;
use Illuminate\Support\Str;

// Use UUID as the resource ID
Resource::resolveIdUsing(fn ($model) => $model->uuid);

// Use snake-cased class name as the type
Resource::resolveTypeUsing(fn ($model) => Str::snake(class_basename($model)));
```

The callback receives the model and the current request:

```php
Resource::resolveIdUsing(function ($model, ?Request $request) {
    return $model->uuid;
});
```

Explicit `$type` on a resource always takes precedence over the global resolver. Reset both resolvers with `Resource::resetResolvers()`.

## Request access

The current request is available as `$this->request` on all Resource methods. It is set automatically when the resource is resolved through `Response`, `QueryBuilder`, or `ResourceCollection`:

```php
public function attributes(Product $product): array
{
    return [
        'name' => $product->name,
        'is_owner' => $this->request?->user()?->is($product->owner),
    ];
}
```

Since the request is a property, you don't need to declare it in every method signature.

## Static `make()` helper

For inline transformations (e.g., nested attributes), use the static `make()` method:

```php
public function attributes(Product $product): array
{
    return [
        'name' => $product->name,
        'price' => MoneyResource::make($product->price_in_cents, $product->currencies),
        'created_at' => DateResource::make($product->created_at),
    ];
}
```
