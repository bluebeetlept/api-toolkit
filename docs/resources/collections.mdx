## How collections work

When you pass an array, `Collection`, or paginator to a success response, the toolkit automatically wraps it in a `ResourceCollection` that:

- Serializes each item using the specified Resource
- Resolves the `included` array from loaded relationships
- Adds pagination `meta` and `links` when applicable

```php
$products = Product::query()->paginate(20);

return $response->success($products, ProductResource::class);
```

## Supported data types

The following types are automatically treated as collections:

- `array`
- `Illuminate\Support\Collection`
- `Illuminate\Database\Eloquent\Collection`
- `Illuminate\Contracts\Pagination\LengthAwarePaginator`
- `Illuminate\Contracts\Pagination\CursorPaginator`

## Pagination metadata

### Offset pagination

When the data is a `LengthAwarePaginator`, the response includes:

```json
{
  "data": [...],
  "meta": {
    "page": {
      "currentPage": 2,
      "lastPage": 5,
      "perPage": 20,
      "total": 100
    }
  },
  "links": {
    "first": "...?page=1",
    "last": "...?page=5",
    "prev": "...?page=1",
    "next": "...?page=3"
  }
}
```

### Cursor pagination

When the data is a `CursorPaginator`:

```json
{
  "data": [...],
  "meta": {
    "page": {
      "perPage": 20,
      "hasMore": true
    }
  },
  "links": {
    "prev": "...?cursor=eyJ...",
    "next": "...?cursor=eyJ..."
  }
}
```

## Included resources

Related resources are automatically deduplicated across all items in the collection:

```json
{
  "data": [
    {
      "type": "products",
      "id": "1",
      "relationships": {
        "category": { "data": { "type": "categories", "id": "5" } }
      }
    },
    {
      "type": "products",
      "id": "2",
      "relationships": {
        "category": { "data": { "type": "categories", "id": "5" } }
      }
    }
  ],
  "included": [
    {
      "type": "categories",
      "id": "5",
      "attributes": { "name": "Gadgets" }
    }
  ]
}
```

Category `5` appears only once in `included`, even though two products reference it.
