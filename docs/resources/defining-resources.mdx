## Defining attributes

The `attributes()` method is the only required method on a Resource. It defines which fields are included in the JSON:API response:

```php
public function attributes(Product $product): array
{
    return [
        'name' => $product->name,
        'code' => $product->code,
        'description' => $product->description,
        'created_at' => $product->created_at->toDateTimeString(),
    ];
}
```

## Type-hinting your model

You can type-hint the actual model class in the first parameter. This gives you full IDE autocompletion and static analysis:

```php
// Full type safety
public function attributes(Product $product): array
{
    return [
        'name' => $product->name, // IDE knows $product is a Product
    ];
}
```

This works because the base `Resource` class uses a generic `@template TModel` and leaves the parameter untyped, allowing child classes to narrow it.

## The request property

The current request is available as `$this->request` on all Resource methods. It is set automatically when the resource is resolved through `Response`, `QueryBuilder`, or `ResourceCollection`:

```php
public function attributes(Product $product): array
{
    return [
        'name' => $product->name,
        'is_owner' => $this->request?->user()?->is($product->owner),
        'internal_notes' => $this->request?->user()?->isAdmin() ? $product->notes : null,
    ];
}
```

Since the request is a property, you don't need to declare it in every method signature.

## Nested resources

Use the static `make()` helper to embed other resources within attributes:

```php
public function attributes(Product $product): array
{
    return [
        'name' => $product->name,
        'price' => MoneyResource::make($product->price_in_cents, $product->currencies),
        'created_at' => DateResource::make($product->created_at),
    ];
}
```

## Resources without a model

Resources don't need to represent Eloquent models. Set `$type` directly and type-hint whatever you need:

```php
final class DateResource extends Resource
{
    protected string $type = 'dates';

    public function attributes(CarbonInterface $date): array
    {
        return [
            'human' => $date->diffForHumans(),
            'string' => $date->toDateTimeString(),
            'local' => $date->toDateTimeLocalString(),
            'timestamp' => $date->timestamp,
        ];
    }
}
```

## Method summary

Every method on a Resource is optional except `attributes()`:

| Method | Receives | Returns | Purpose |
|--------|----------|---------|---------|
| `attributes()` | model | `array` | **Required.** The resource's data fields |
| `self()` | model | `string\|null` | The self link URL |
| `links()` | model | `array` | Additional links |
| `meta()` | model | `array` | Resource-level metadata |
| `relationships()` | -- | `array` | Relationship map |
| `allowedFilters()` | -- | `array` | QueryBuilder filter config |
| `allowedSorts()` | -- | `array` | QueryBuilder sort config |
| `defaultSort()` | -- | `string\|null` | QueryBuilder default sort |
| `allowedIncludes()` | -- | `array` | QueryBuilder include config |

All methods have access to `$this->request` for request context.
