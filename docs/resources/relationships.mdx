## Defining relationships

The `relationships()` method maps relationship names to their corresponding Resource classes:

```php
public function relationships(): array
{
    return [
        'category' => CategoryResource::class,
        'supplier' => SupplierResource::class,
        'tags' => TagResource::class,
    ];
}
```

## How relationships are resolved

Relationships are only included in the response when they are **eager-loaded** on the Eloquent model. If a relationship is not loaded, it is omitted from the response.

### Single relationship (BelongsTo, HasOne)

```json
{
  "relationships": {
    "category": {
      "data": { "type": "categories", "id": "5" }
    }
  }
}
```

### Collection relationship (HasMany, BelongsToMany)

```json
{
  "relationships": {
    "tags": {
      "data": [
        { "type": "tags", "id": "1" },
        { "type": "tags", "id": "2" }
      ]
    }
  }
}
```

### Null relationship

```json
{
  "relationships": {
    "category": {
      "data": null
    }
  }
}
```

## Including related resources

When a relationship is included via `?include=category`, the full resource object appears in the top-level `included` array:

```bash
GET /products?include=category
```

```json
{
  "data": [
    {
      "type": "products",
      "id": "1",
      "attributes": { "name": "Widget" },
      "relationships": {
        "category": {
          "data": { "type": "categories", "id": "5" }
        }
      }
    }
  ],
  "included": [
    {
      "type": "categories",
      "id": "5",
      "attributes": { "name": "Gadgets" }
    }
  ]
}
```

<Note>
  The `included` array is deduplicated. Each resource appears only once, even if referenced by multiple items in `data`.
</Note>

## Conditional relationships

Since `$this->request` is available on all methods, you can conditionally expose relationships:

```php
public function relationships(): array
{
    $relationships = [
        'category' => CategoryResource::class,
    ];

    if ($this->request?->user()?->isAdmin()) {
        $relationships['auditLog'] = AuditLogResource::class;
    }

    return $relationships;
}
```

## Allowed includes

By default, `allowedIncludes()` returns the keys of `relationships()`. This means all defined relationships can be included. Override it to restrict:

```php
public function allowedIncludes(): array
{
    // 'supplier' is a relationship but cannot be included via ?include
    return ['category'];
}
```
