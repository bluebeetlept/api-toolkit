# Includes

The JSON:API `include` parameter lets clients request related resources in the same response, avoiding N+1 requests.

## Defining Allowed Includes

By default, `allowedIncludes()` returns the keys from your `relationships()` definition:

```php
class OrderResource extends Resource
{
    public function relationships(): array
    {
        return [
            'customer' => CustomerResource::class,
            'items'    => OrderItemResource::class,
        ];
    }

    // allowedIncludes() defaults to ['customer', 'items']
}
```

Override it to restrict which relationships can be included:

```php
public function allowedIncludes(): array
{
    return ['customer']; // items cannot be included
}
```

## Usage

```
GET /api/orders?include=customer
GET /api/orders?include=customer,items
GET /api/orders?include=items.product         → nested include
```

## Response Structure

When includes are requested, related resources appear in the `included` array:

```json
{
  "data": {
    "type": "orders",
    "id": "1",
    "attributes": { "total": 150.00 },
    "relationships": {
      "customer": {
        "data": { "type": "customers", "id": "42" }
      }
    }
  },
  "included": [
    {
      "type": "customers",
      "id": "42",
      "attributes": { "name": "John Doe" }
    }
  ]
}
```

Included resources are automatically deduplicated — each unique type/id pair appears only once, even if multiple records reference the same related resource.

## Validation

Use the `ValidInclude` rule in form requests:

```php
use BlueBeetle\ApiToolkit\Rules\ValidInclude;

public function queryParamRules(): array
{
    return [
        'include' => ['sometimes', new ValidInclude(['customer', 'items'])],
    ];
}
```
