# Pagination

The toolkit supports two pagination strategies: offset-based and cursor-based.

## Offset Pagination

Traditional page-number pagination. Use `->paginate()` on the query builder.

```php
return QueryBuilder::for(Product::class, $request)
    ->fromResource(ProductResource::class)
    ->paginate();
```

### Query Parameters

```
GET /api/products?page[number]=2&page[size]=20
```

### Response Meta

```json
{
  "data": [...],
  "meta": {
    "page": {
      "currentPage": 2,
      "lastPage": 5,
      "perPage": 20,
      "total": 100
    }
  },
  "links": {
    "first": "...",
    "last": "...",
    "prev": "...",
    "next": "..."
  }
}
```

## Cursor Pagination

More efficient for large datasets. Use `->cursorPaginate()`.

```php
return QueryBuilder::for(Product::class, $request)
    ->fromResource(ProductResource::class)
    ->cursorPaginate();
```

### Query Parameters

```
GET /api/products?page[size]=20
GET /api/products?page[cursor]=eyJpZCI6MjB9&page[size]=20
```

### Response Meta

```json
{
  "data": [...],
  "meta": {
    "page": {
      "perPage": 20,
      "hasMore": true
    }
  },
  "links": {
    "prev": "...",
    "next": "..."
  }
}
```

The cursor value is opaque to the client — they should use it as-is from the `links.next` URL.

## Configuration

In `config/api-toolkit.php`:

```php
'pagination' => [
    'default_size' => 20,     // Default items per page
    'max_size'     => 100,    // Maximum items per page (requests above this are capped)
    'valid_sizes'  => [10, 20, 40, 80, 100],
],
```

If a client requests a page size greater than `max_size`, it's silently capped. A size below 1 is clamped to 1.

## Validation

Use the `ValidPageSize` rule in form requests to validate the page size:

```php
use BlueBeetle\ApiToolkit\Rules\ValidPageSize;

public function queryParamRules(): array
{
    return [
        'page.size' => ['sometimes', new ValidPageSize()],
    ];
}
```

By default, it accepts the sizes defined in the config. Pass custom sizes to the constructor:

```php
new ValidPageSize([5, 10, 25, 50])
```
