# Filters

Filters allow API consumers to narrow down results using query parameters. The JSON:API convention uses the format `?filter[field]=value`.

## Built-in Filters

### ExactFilter

Matches an exact value. Supports arrays for `WHERE IN` queries.

```php
use BlueBeetle\ApiToolkit\Parsers\Filters\ExactFilter;

public function allowedFilters(): array
{
    return [
        'status' => new ExactFilter(),
    ];
}
```

```
GET /api/products?filter[status]=active
GET /api/products?filter[status]=active,archived   → WHERE status IN ('active', 'archived')
```

### PartialFilter

Performs a `LIKE %value%` search.

```php
use BlueBeetle\ApiToolkit\Parsers\Filters\PartialFilter;

public function allowedFilters(): array
{
    return [
        'name' => new PartialFilter(),
    ];
}
```

```
GET /api/products?filter[name]=widget   → WHERE name LIKE '%widget%'
```

### DateFilter

Filters by date. Supports exact dates and date ranges using `from` and `to`.

```php
use BlueBeetle\ApiToolkit\Parsers\Filters\DateFilter;

public function allowedFilters(): array
{
    return [
        'created_at' => new DateFilter(),
    ];
}
```

```
GET /api/products?filter[created_at]=2025-01-15
GET /api/products?filter[created_at][from]=2025-01-01&filter[created_at][to]=2025-01-31
```

### ScopeFilter

Delegates filtering to a query scope on the model.

```php
use BlueBeetle\ApiToolkit\Parsers\Filters\ScopeFilter;

public function allowedFilters(): array
{
    return [
        'category' => new ScopeFilter(),
    ];
}
```

This calls `$query->category($value)` on the Eloquent builder, so your model needs a matching scope:

```php
class Product extends Model
{
    public function scopeCategory(Builder $query, string $value): void
    {
        $query->whereHas('category', fn ($q) => $q->where('slug', $value));
    }
}
```

To use a scope name that differs from the filter key, pass it to the constructor:

```php
'active' => new ScopeFilter('whereActive'),
```

This calls `$query->whereActive($value)` instead of `$query->active($value)`.

## Custom Filters

Implement the `Filter` interface to create your own:

```php
use BlueBeetle\ApiToolkit\Parsers\Filters\Filter;
use Illuminate\Database\Eloquent\Builder;

class PriceRangeFilter implements Filter
{
    public function apply(Builder $query, string $field, mixed $value): void
    {
        if (is_array($value)) {
            $query->when(isset($value['min']), fn ($q) => $q->where($field, '>=', $value['min']))
                  ->when(isset($value['max']), fn ($q) => $q->where($field, '<=', $value['max']));
        }
    }
}
```

Use it:

```php
public function allowedFilters(): array
{
    return [
        'price' => new PriceRangeFilter(),
    ];
}
```

```
GET /api/products?filter[price][min]=10&filter[price][max]=50
```

## Unknown Filters

Any filter not listed in `allowedFilters()` is silently ignored. This prevents errors when consumers send unsupported filters.
