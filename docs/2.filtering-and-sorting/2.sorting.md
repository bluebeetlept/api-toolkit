# Sorting

Sorting follows the JSON:API convention: `?sort=field1,-field2` where a `-` prefix means descending order.

## Defining Allowed Sorts

In your resource:

```php
class ProductResource extends Resource
{
    public function allowedSorts(): array
    {
        return ['name', 'price', 'created_at'];
    }

    public function defaultSort(): ?string
    {
        return '-created_at'; // newest first
    }
}
```

## Usage

```
GET /api/products?sort=name                    → ORDER BY name ASC
GET /api/products?sort=-price                  → ORDER BY price DESC
GET /api/products?sort=name,-created_at        → ORDER BY name ASC, created_at DESC
GET /api/products                              → ORDER BY created_at DESC (default)
```

## Behavior

- Sort fields not in `allowedSorts()` are silently ignored
- If no `?sort` parameter is provided and a `defaultSort()` is defined, it's applied automatically
- Multiple sort fields are applied in the order specified
