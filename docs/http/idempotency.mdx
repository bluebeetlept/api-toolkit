---
title: Idempotency
description: Built-in idempotency support for API requests
---

API Toolkit includes the [`bluebeetlept/idempotency-middleware`](https://github.com/bluebeetlept/idempotency-middleware) package, which provides idempotency support for your API endpoints.

## How it works

Clients send an `Idempotency-Key` header (a UUID v4) with their request. If the same key is sent again with the same body and endpoint, the cached response is returned instead of re-executing the request. The `Idempotent-Replayed` header is added to indicate a replayed response.

## Usage

Apply the middleware to routes that should support idempotency:

```php
use BlueBeetle\IdempotencyMiddleware\Idempotency;

Route::middleware(Idempotency::class)->group(function () {
    Route::post('/products', CreateController::class);
    Route::post('/products/{product}', UpdateController::class);
});
```

## Client usage

```http
POST /api/v1/products HTTP/1.1
Content-Type: application/vnd.api+json
Idempotency-Key: 6b3fd36c-24c6-4eb2-a764-bb6c91b33e56

{"name": "Widget", "code": "W01"}
```

If the same `Idempotency-Key` is sent again with the same body and endpoint, the original response is returned from cache.

## Configuration

Publish the config file:

```bash
php artisan vendor:publish --tag="bluebeetle-idempotency-config"
```

Or configure via environment variables:

```dotenv
IDEMPOTENCY_MAIN_HEADER="X-Idempotency-Key"
IDEMPOTENCY_REPEATED_HEADER="X-Idempotent-Replayed"
IDEMPOTENCY_EXPIRATION=720
```

| Variable | Default | Description |
|----------|---------|-------------|
| `IDEMPOTENCY_MAIN_HEADER` | `Idempotency-Key` | Header name for the idempotency key |
| `IDEMPOTENCY_REPEATED_HEADER` | `Idempotent-Replayed` | Header added to replayed responses |
| `IDEMPOTENCY_EXPIRATION` | `720` | Cache TTL in minutes (default: 12 hours) |

## Error handling

The exception handler automatically converts `IdempotencyException` into a JSON:API error response. This happens when:

- The idempotency key is not a valid UUID v4
- The key was used before with a different request body
- The key was used before with a different endpoint

```json
{
  "errors": [
    {
      "status": "400",
      "code": "idempotency_error",
      "title": "Idempotency Error",
      "detail": "The idempotency key was already used with a different request body."
    }
  ]
}
```
