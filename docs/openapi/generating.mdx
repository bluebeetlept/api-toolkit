## Basic usage

```bash
php artisan api-toolkit:openapi
```

This generates `openapi.json` in your project root.

## Command options

| Option | Default | Description |
|--------|---------|-------------|
| `--output` | `openapi.json` | Output file path |
| `--pretty` | _(off)_ | Pretty-print the JSON |

```bash
php artisan api-toolkit:openapi --pretty --output=public/openapi.json
```

## Configuration

All OpenAPI settings live in `config/api-toolkit.php` under the `openapi` key:

```php config/api-toolkit.php
'openapi' => [
    'title' => env('APP_NAME', 'API'),
    'version' => '1.0.0',
    'description' => 'REST API for managing resources',

    'servers' => [
        ['url' => env('APP_URL', 'http://localhost') . '/api'],
    ],

    'security_schemes' => [
        'bearerAuth' => [
            'type' => 'http',
            'scheme' => 'bearer',
        ],
    ],

    'security' => [
        ['bearerAuth' => []],
    ],
],
```

### Servers

Define one or more API server URLs:

```php
'servers' => [
    ['url' => 'https://api.example.com', 'description' => 'Production'],
    ['url' => 'https://staging-api.example.com', 'description' => 'Staging'],
],
```

### Security schemes

Define authentication methods. Common examples:

```php
// Bearer token
'security_schemes' => [
    'bearerAuth' => [
        'type' => 'http',
        'scheme' => 'bearer',
    ],
],

// OAuth2
'security_schemes' => [
    'oauth2' => [
        'type' => 'oauth2',
        'flows' => [
            'clientCredentials' => [
                'tokenUrl' => 'https://auth.example.com/oauth/token',
                'scopes' => [
                    'read' => 'Read access',
                    'write' => 'Write access',
                ],
            ],
        ],
    ],
],
```

### Global security

Apply security globally to all endpoints:

```php
'security' => [
    ['bearerAuth' => []],
],
```

## Route detection

The generator scans all registered routes and identifies those that:
- Return responses via `Response::success()` with a Resource class
- Use `QueryBuilder::for()` with a Resource class

It detects the Resource class from the controller source code by looking for `SomeResource::class` references.

## What gets generated

For each detected endpoint:

| Element | Source |
|---------|--------|
| Path and HTTP method | Laravel route |
| Operation ID | Route name |
| Summary | Auto-generated from Resource name and HTTP method |
| Tags | Resource name (pluralized) |
| Path parameters | Route parameters with descriptions |
| Query parameters | Resource's `allowedFilters()`, `allowedSorts()`, `allowedIncludes()` |
| Request body | FormRequest `rules()` |
| Success response | Resource schema with JSON:API envelope |
| Error responses | Shared `$ref` components (401, 404, 422) |

## Attribute types

The generator resolves attribute types in this order:

### 1. Explicit schema (highest priority)

Override the `schema()` method on your Resource for full control:

```php
public function schema(): array
{
    return [
        'name' => 'string',
        'quantity' => 'integer',
        'active' => 'boolean',
        'created_at' => 'datetime',
        'price' => ['type' => 'number', 'format' => 'float', 'example' => 9.99],
    ];
}
```

Shorthand types: `string`, `integer`, `int`, `number`, `float`, `boolean`, `bool`, `array`, `object`, `date`, `datetime`.

You can also pass full OpenAPI property definitions for complex cases.

### 2. Model casts (automatic)

When `$model` is set and no `schema()` is defined, the generator reads the model's `$casts` array:

```php
// On your model
protected $casts = [
    'quantity' => 'integer',
    'active' => 'boolean',
    'price' => 'decimal:2',
    'status' => ProductStatus::class,  // PHP enum
    'created_at' => 'datetime',
];
```

Cast mappings:
| Cast | OpenAPI type |
|------|-------------|
| `integer`, `int` | `integer` |
| `float`, `double`, `decimal`, `decimal:N` | `number` |
| `boolean`, `bool` | `boolean` |
| `string` | `string` |
| `date` | `string` (format: `date`) |
| `datetime`, `immutable_datetime` | `string` (format: `date-time`) |
| `array`, `json`, `collection` | `object` |
| PHP BackedEnum | `string` or `integer` with `enum` values |

### 3. Fallback

If neither `schema()` nor model casts are available, the attributes schema uses `additionalProperties: true` (any object).

## Request body generation

For POST/PUT/PATCH endpoints, the generator reads validation rules from the controller's FormRequest:

```php
public function rules(): array
{
    return [
        'name' => ['required', 'string', 'min:3'],
        'email' => ['nullable', 'email'],
        'quantity' => ['required', 'integer'],
    ];
}
```

This produces:

```json
{
  "type": "object",
  "required": ["name", "quantity"],
  "properties": {
    "name": {"type": "string", "minLength": 3},
    "email": {"type": "string", "format": "email", "nullable": true},
    "quantity": {"type": "integer"}
  }
}
```

## Shared error responses

Common error responses are registered as `$ref` components and reused across all endpoints:

- `#/components/responses/UnauthorizedException` (401)
- `#/components/responses/NotFoundHttpException` (404)
- `#/components/responses/ValidationException` (422)

All follow the JSON:API error format.

## CI integration

Add the generator to your CI pipeline to keep the spec in sync:

```yaml
- name: Generate OpenAPI spec
  run: php artisan api-toolkit:openapi --output=public/openapi.json

- name: Validate OpenAPI spec
  run: npx @redocly/cli lint public/openapi.json
```
